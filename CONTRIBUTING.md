# Contribuir a circuit

¡Gracias por querer contribuir! Aquí tienes la guía rápida para colaborar de forma eficiente.

## Formas de contribuir
- Reportar issues (bugs, ideas, solicitudes)
- Escribir código (features, correcciones)
- Crear adaptadores a otras herramientas/simuladores
- Añadir componentes a la librería
- Mejorar documentación y ejemplos

## Flujo recomendado (git)
1. Haz fork del repositorio (si no tienes permisos de push).
2. Crea una rama descriptiva: `feature/nueva-funcionalidad` o `fix/corrección`.
3. Haz commits atómicos y descriptivos (convención: feat, fix, docs).
4. Abre un Pull Request contra `main` (o la rama base indicada).
5. Responde a las revisiones y actualiza hasta la aprobación.

## Estilo de código
- Especificaremos linters por lenguaje (añadir en futuras tareas).
- Añade tests para cambios importantes o fixes.

## Documentación técnica
- Cada adaptador/plugin debe incluir un README con: propósito, instalación, ejemplos y API.

## Issues y PRs
- Al abrir un issue: título claro, pasos para reproducir y ejemplo si aplica.
- Al abrir un PR: describe motivación, cambios y cómo probarlos.

## Comunicación
- Usaremos GitHub Issues y Discussions para coordinación.
- Podemos integrar un canal en Discord o Matrix si la comunidad lo solicita.

Gracias por tu contribución — cada aporte cuenta.
